Node.js · Express · TypeScript · MongoDB

Production-style REST API for a publishing platform.

A portfolio-ready backend with versioned routes, session authentication, role-based authorization, Zod validation, security middleware, and a consistent JSON response contract.

Security

Defense in depth

Helmet headers, CORS, rate limiting, bcrypt hashing, and sanitized error responses.

Architecture

Layered design

Routes, controllers, validators, models, and middleware are separated for clarity and testing.

Features

Publishing workflow

Admin post management, public reads, comments, likes, view counts, and pagination.

Core Routes

  • POST /api/v1/users/registerRegister user
  • POST /api/v1/users/loginLog in
  • GET /api/v1/users/meCurrent session
  • GET /api/v1/postsList posts
  • POST /api/v1/postsCreate post (admin)
  • POST /api/v1/posts/:id/commentsAdd comment
